linux 开发板 如何设置时间并同步到RTC

发表于 2025-11-10 分类于 技术分享 ， Linux

在 Linux 系统中（尤其是在开发板上），设置时间并确保它能在断电后被保存，涉及到两个时钟：软件时钟和硬件时钟 (RTC)。

我将为您分步解释如何操作，并提供最佳的“一劳永逸”的自动化方案。

理解两个时钟

首先，您必须理解系统中有两个时钟：

  1. 系统时钟 (System Clock / 软件时钟): 这是由 Linux 内核维护的时钟。您在终端里用 date 命令看到的就是它。它在系统运行时非常精确，但系统一旦断电，它的时间就会丢失。下次开机时，它需要一个初始值。

  2. 硬件时钟 (Hardware Clock / RTC / 硬件时钟): 这是一个独立的物理芯片，由电池供电。即使开发板断电，它也能继续走时。它的主要作用就是在系统开机时，为“系统时钟”提供一个准确的初始时间。您用 hwclock 命令操作的就是它。

我们的目标是：

  1. 设置一个准确的系统时钟。
  2. 将这个准确的时间从系统时钟同步（保存）到硬件时钟。
  3. 配置系统，让这个过程尽可能自动化。

方法一：手动设置并保存 (适用于无网络环境)

如果您无法连接互联网，或者需要设置一个特定的时间，可以使用此方法。

步骤 1: 设置系统时钟

使用 date 命令来设置当前的系统时间。格式为 月日时分年.秒 或者更易读的格式。

方法二：自动设置并保存 (推荐的最佳实践)

手动设置很麻烦，而且时钟总会有误差。最好的方法是让系统连接互联网，通过 NTP (网络时间协议) 自动同步时间，并让系统自动将时间写入 RTC。

现代 Linux 系统（使用 systemd 的，如 Ubuntu, Debian, Fedora 等）大多内置了 systemd-timesyncd 服务，可以非常方便地实现这一点。

关键点解读：

  • System clock synchronized: yes: 这表明系统时钟已经通过网络成功同步。
  • NTP service: active: 表示时间同步服务正在运行。
  • RTC time: 这是硬件时钟的时间。注意，它通常被设置为 UTC 时间，这是标准做法。系统会根据您的时区设置自动换算成本地时间。

它是如何“自动保存”的？

当您启用了 systemd-timesyncd 后，systemd 框架本身会负责处理后续的事情：

  1. 它会定期通过网络校准系统时钟。
  2. 在系统正常关机或重启时，一个名为 systemd-hwclock-commit.service 的服务会自动运行，将准确的系统时钟写入硬件时钟 (RTC)。

所以，您只需要执行 sudo timedatectl set-ntp true 这一条命令，系统就会实现时间的自动获取和自动保存，一劳永逸。

总结

| 需求场景 | 推荐方法 | 核心命令 - |
| 无网络环境，需要手动设置 | 方法一：手动设置 | 1. sudo date -s "YYYY-MM-DD HH:MM:SS" (设置系统时间)
2. sudo hwclock --systohc (保存到硬件) - |
| 有网络环境，希望自动同步 | 方法二：NTP 自动同步 (推荐) | 1. sudo timedatectl set-timezone "Asia/Shanghai" (设置时区)
2. sudo timedatectl set-ntp true (开启NTP) - |
